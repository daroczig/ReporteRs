Features

  • Insert paragraph of texts into the output document.
  • Apply formating properties to chunk of texts

Usage

addParagraph require one of the following two types of argument:

  • Either a set_of_paragraphs object
  • Either a character vector

A set_of_paragraphs object is a container for pot objects:

  • A pot object is a paragraph of texts, each text has its own font properties.
  • a pot object is a convenient way to define a paragraph of text where some texts are not all formated the same.
  • pot object can be added with another pot object or with a single text value (a character vector of length 1).

pot object can be created with the function pot.

The function has two arguments:

  • value: a text value or a value that has a format method returning character value.
  • format (optional): formating properties (an object returned by textProperties).

Below some examples: 

# create a pot made of "My tailor" with a red font.
text = pot("My tailor", textProperties(color="#354B5E", font.weight = "bold", font.size = 20 ) )
doc = addParagraph( doc, set_of_paragraphs(text), stylename = "normal" )

# a quite  full demo.
text = pot( value = "My tailor"
        , format = textProperties(color="#354B5E"
                                , font.size = 20
                                , font.weight = "bold"
                                , font.style = "italic"
                                , underlined = TRUE
                                , font.family = "Courier New"
                )
) + " is rich"
doc = addParagraph( doc, set_of_paragraphs(text), stylename = "normal" )

It produces the following result:

My tailor

My tailor is rich

options("ReporteRs-fontsize"=20)

addParagraph behaviours

If addParagraph is used with a docx object, default font properties will be those defined in the style defined by argument stylename.

If beeing used with a pptx object, default font properties will be those defined in the corresponding shape (of the used layout)

If beeing used with an html object, default font properties will be those defined by argument stylename (this need to be defined somewhere as a css properties declaration, if not properties will be 'body' properties.).
